主題色規劃

三層 token 架構，換膚只改一處

現有的 8 組主題已經是運作中的 CSS 變數系統，不需重做。規劃重點是把顏色明確分成原始色階 → 語意 token → 元件 token 三層，並補上散落硬寫的狀態色、文字階層，以及全新的直播專用色。點下方色票即時預覽換膚效果。

② 三層 token 架構

層級角色範例隨主題變
L1 原始色階
Primitive		純色票，只是調色盤，不帶語意--purple-500: #7008e7否
L2 語意 token
Semantic		描述「用途」，指向 L1 或主題值--color-primary --color-danger部分
L3 元件 token
Component		元件內部專用，指向 L2，多用 color-mix 衍生--btn-primary-bg --live-chat-bg否（衍生）

落地原則：元件只引用 L2 / L3，不直接寫死色碼。換膚只覆寫少數 L2 key，其餘自動衍生（現有 --icon-primary-* 已是這模式，延用）。
